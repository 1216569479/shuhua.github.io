webpack之基础概念

1. webpack简介(https://www.webpackjs.com/)

webpack is a module bundler(模块打包⼯工具)
Webpack是⼀个打包模块化JavaScript的⼯具，它会从⼊口模块出发，识别出源码中的模块化导⼊语句，递归地找出入⼝⽂件的所有依赖，将⼊口和其所有的依赖打包到⼀个单独的⽂件中,是⼯程化、自动化思想在前端开发中的体现

2.安装webpack

2.1-环境准备

nodeJs https://nodejs.org/en/ （node需要多版本，考虑使用nvm）
版本参考官⽹网发布的最新版本，可以提升webpack的打包速度

2.2-全局安装（不推荐）

# 安装webpack V4+版本时，需要额外安装webpack-cli npm install webpack webpack-cli -g 
# 检查版本 webpack -v
# 卸载 npm uninstall webpack webpack-cli -g

全局安装webpack，这会将你项⽬目中的webpack锁定到指定版本，造成不不 同的项⽬目中因为webpack依赖不不同版本⽽而导致冲突，构建失败

2.3-项⽬目安装 (推荐 )

# 安装最新的稳定版本 npm i -D webpack
# 安装指定版本 npm i -D webpack@<version>
# 安装最新的体验版本 可能包含bug,不要⽤用于⽣生产环境 npm i -D webpack@beta
# 安装webpack V4+版本时，需要额外安装webpack-cli npm i -D webpack-cli

####2.4-检查安装

webpack -v //command not found 默认在全局环境中查找
npx webpack -v// npx帮助我们在项⽬目中的node_modules⾥里里查找 webpack ./node_modules/.bin/webpack -v//到当前的node_modules模 块⾥里里指定webpack

3.启动webpack执⾏行行构建

1Chunk = 1bundle(chunk是代码块,bundle是资源⽂文件)
1个chunk可以是多个模块组成的
模块？nodeJS⾥,万物皆模块
数组：webpack会⾃自动⽣生成另外⼀个⼊⼝模块，并将数组中的每个指定的 模块加载进来，并将最后⼀个模块的module.exports作为⼊⼝模块 的module.exports导出

3.1- webpack默认配置

  • webpack默认支持JS模块和JSON模块
  • 支持CommonJS Es moudule AMD等模块类型
  • webpack4支持零配置使⽤用,但是很弱，稍微复杂些的场景都需要 额外扩展

3.2- 准备执行构建

  • 新建src⽂文件夹
  • 新建src/index.js、src/index.json、src/other.js
### index.js 
const json = require("./index.json");//commonJS 
import &#123; add &#125; from "./other.js";//es module
console.log(json, add(2, 3));
### index.json
&#123;  "name": "JOSN" &#125;
### other.js 
export function add(n1, n2) &#123;  return n1 + n2; &#125;

3.3- 执行构建

# npx⽅方式 
npx webpack
# npm script 
npm run test

修改package.json文件：

"scripts": &#123;  "test": "webpack" &#125;,

原理就是通过shell脚本在node_modules/.bin目录下创建一个软链接。

3.4-构建成功

我们会发现目录下多出一个dist目录，里面有个main.js，这个文件是一个可执行的JavaScript文件，里面包含webpackBootstrap 启动函数。

3.5-默认配置

const path = require("path"); 
module.exports = &#123;  
    // 必填  webpack执⾏行行构建⼊⼝口 
    entry: "./src/index.js", 
    output: &#123;    
     //  将所有依赖的模块合并输出到main.js   
    filename: "main.js",   
     //  输出⽂件的存放路径，必须是绝对路径   
    path: path.resolve(__dirname, "./dist") 
    &#125;

4.webpack配置核心概念

webpack有默认的配置文件，叫webpack.config.js，我们可以 对这个文件进行修改，进行个性化配置

  • 使用默认的配置文件：webpack.conﬁg.js
  • 不使用自定义配置文件： 比如webpackconﬁg.js，可以通过-conﬁg webpackconﬁg.js来指定webpack使用哪个配置文件来 执行构建

webpack.conﬁg.js配置基础结构

module.exports = &#123;  
 entry: "./src/index.js", //打包⼊入⼝口⽂文件  
 output: "./dist", //输出结构 
 mode: "production", //打包环境 
 module: &#123;  
    rules: [      
    //loader模块处理理    
    &#123;  test: /\.css$/, use: "style-loader"      &#125; 
    ] 
 &#125;, 
 plugins: [new HtmlWebpackPlugin()] //插件配置
